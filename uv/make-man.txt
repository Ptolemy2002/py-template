MAKEFILE REFERENCE
==================

This Makefile provides targets for managing a Python project. The script
must be run through `make start` (not directly via python/uv) because utilities
expect the output directory structure created by the start target.

QUICK START
-----------

  make start ARGS="your arguments here"    Run main.py with optional arguments
  make help ARGS="extra flags"             Show main.py help message
  make reset                                Clean everything and start fresh

MAIN TARGETS
------------

start
  Creates a timestamped output directory and runs main.py, capturing all output
  to a log file. The log is cleaned of ANSI color codes for readability.
  
  Example: make start ARGS="--verbose --input data.txt"
  
  Output location: src/outputs/main/YYYY/MM/DD/HH-MM-SS/log.txt

help
  Shows main.py's help message. You can pass additional flags via ARGS.
  
  Example: make help ARGS="--advanced"

make-help
  Displays this help message.

reset
  Nuclear option: removes all generated files, caches, lock files, and virtual
  environments. Use this when things are broken or you want a clean slate.
  Runs: rmzi, rm-lock, rm-outputs, rm-pycache, rm-venv

CLEANUP TARGETS
---------------

rmzi
  Remove Zone Identifier files (those annoying ":Zone.Identifier" files that
  appear when copying from Windows to WSL).

rm-lock
  Deletes uv.lock to force dependency resolution from scratch.

rm-outputs
  Removes the entire src/outputs/ directory and all logs.

rm-pycache
  Cleans Python bytecode cache directories recursively.

rm-venv
  Removes the .venv virtual environment.

SETUP TARGETS
-------------

dep-install
  Installs system dependencies (colorized-logs and moreutils). Requires apt/sudo.

uv-install
  Installs the uv package manager via official installer script.


ARGUMENTS
---------

Pass arguments to main.py using ARGS variable:

  make start ARGS="--config settings.json --debug"
  make help ARGS="--advanced-options"

The help target automatically prepends --help to whatever you pass in ARGS.


NOTES
-----

- All timestamps use your local timezone
- Log files contain both stdout and stderr
- ANSI color codes are stripped from final log files for easier parsing
- Don't run main.py directly; always use `make start`
